# CI Checks Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a `ci.yml` GitHub Actions workflow that runs compile, format, credo, and test checks on every push to `main` and every pull request.

**Architecture:** Single workflow file with one job (`test`) running on `ubuntu-latest` with a `postgres:16` service container. Steps mirror `mix precommit` but split into individual steps for visibility and exclude the `deps.unlock --unused` mutation. Deps and build artifacts are cached keyed on `mix.lock` hash to keep runs fast.

**Tech Stack:** GitHub Actions, `erlef/setup-beam@v1`, `actions/cache@v4`, Elixir 1.19.2 / OTP 28, PostgreSQL 16, ExUnit, Credo

### Task 1: Create the CI workflow file

**Files:**
- Create: `.github/workflows/ci.yml`

- [ ] **Step 1: Create the workflow file**

Create `.github/workflows/ci.yml` with this exact content:

```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:

jobs:
  test:
    name: Build and Test
    runs-on: ubuntu-latest

    services:
      postgres:
        image: postgres:16
        env:
          POSTGRES_USER: postgres
          POSTGRES_PASSWORD: postgres
          POSTGRES_DB: fountain_test
        ports:
          - 5432:5432
        options: >-
          --health-cmd pg_isready
          --health-interval 10s
          --health-timeout 5s
          --health-retries 5

    env:
      MIX_ENV: test

    steps:
      - uses: actions/checkout@v4

      - name: Set up Elixir
        uses: erlef/setup-beam@v1
        with:
          elixir-version: "1.19.2"
          otp-version: "28"

      - name: Restore deps cache
        uses: actions/cache@v4
        with:
          path: deps
          key: ${{ runner.os }}-mix-${{ hashFiles('**/mix.lock') }}
          restore-keys: |
            ${{ runner.os }}-mix-

      - name: Restore build cache
        uses: actions/cache@v4
        with:
          path: _build
          key: ${{ runner.os }}-build-28-1.19.2-${{ hashFiles('**/mix.lock') }}
          restore-keys: |
            ${{ runner.os }}-build-28-1.19.2-
            ${{ runner.os }}-build-

      - name: Install dependencies
        run: mix deps.get

      - name: Compile (warnings as errors)
        run: mix compile --warnings-as-errors

      - name: Check formatting
        run: mix format --check-formatted

      - name: Run Credo
        run: mix credo --strict

      - name: Set up database
        run: mix ecto.create --quiet && mix ecto.migrate --quiet

      - name: Run tests
        run: mix test
```

- [ ] **Step 2: Commit the workflow file**

```bash
git add .github/workflows/ci.yml
git commit -m "ci: add CI workflow for compile, format, credo, and test checks"
```

### Task 2: Verify CI triggers and passes

- [ ] **Step 1: Push the branch and open a PR**

Push to a feature branch (e.g. `ci/add-checks`) and open a PR against `main`. The new `CI / Build and Test` check should appear within ~30 seconds of the PR being opened.

- [ ] **Step 2: Confirm all steps pass**

In the PR checks panel, expand the `CI / Build and Test` job and confirm each step completes successfully:
- `Set up Elixir` — installs OTP 28 + Elixir 1.19.2
- `Install dependencies` — `mix deps.get` exits 0
- `Compile (warnings as errors)` — exits 0
- `Check formatting` — exits 0
- `Run Credo` — exits 0
- `Set up database` — creates and migrates `fountain_test`
- `Run tests` — all tests pass

- [ ] **Step 3: Confirm cache is populated**

On a second CI run (re-run the job or push a fixup commit), confirm the cache restore steps show a cache hit. You should see `Cache restored successfully` for both `deps` and `_build` caches, and `Install dependencies` should be noticeably faster (~seconds vs ~minutes).

- [ ] **Step 4: Smoke-test failure detection**

Add a deliberate formatting issue to any Elixir file (e.g. add an extra blank line in `apps/fountain/lib/fountain/application.ex`), push to the same branch, and verify the `Check formatting` step fails with:

```
** (Mix) mix format failed due to --check-formatted.
The following files are not formatted:
  apps/fountain/lib/fountain/application.ex
```

Then revert the change and push again to confirm CI goes green.

```bash
# revert the deliberate change
git revert HEAD --no-edit
git push
```

- [ ] **Step 5: Merge the PR**

Once CI is green, merge the PR. From this point forward every PR will have automated quality gates.

## Acceptance Criteria Checklist

- [ ] `.github/workflows/ci.yml` exists
- [ ] Workflow runs on push to `main` and on pull requests
- [ ] A PR with a compilation warning fails the `Compile` step
- [ ] A PR with misformatted code fails the `Check formatting` step
- [ ] A PR with a Credo violation fails the `Run Credo` step
- [ ] A PR with a failing test fails the `Run tests` step
- [ ] `deps/` and `_build/` are cached between runs
- [ ] No repository secrets are required for the workflow to run
